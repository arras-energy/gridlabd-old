HiPAS GridLAB-D Online Installation & Validation

Installing HiPAS GridLAB-D

IMPORTANT: This installer is designed to install HiPAS GridLAB-D on a new system image such as wsl, ec2, or docker. Do not use this on an existing system unless you are ok with significant changes to your system's build tools.

To install HiPAS GridLAB-D on your Linux or Mac system, type the following commands at a shell prompt:

bash$ curl -sL http://install.gridlabd.us/install.sh | sh If your system is not supported by this installer, please use the manual installation method provided with the HiPAS GridLAB-D source code.

Validating HiPAS GridLAB-D

To validate your HiPAS GridLAB-D installation on your Linux or Mac system, type the following command at a shell prompt:

bash$ git clone https://source.gridlabd.us/ gridlabd bash$ gridlabd -W gridlabd --validate

The validation process takes around 10 minutes.

The result will be placed in a folder called validate.txt. Errors will be reported as it goes. To inspect an error, find the output files in the folder of the same name as the GLM file, i.e., gridlabd/PATH/autotest/FILE.glm will place output files in gridlabd/PATH/autotest/FILE
